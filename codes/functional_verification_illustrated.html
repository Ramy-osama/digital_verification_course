Simulation-Based Verification
Apply stimulus to DUT, observe and check outputs via waveforms
■ RTL Simulation Waveforms
Typical simulation waveform: clock, reset, data bus transitions with 1-cycle pipeline delay
■ Directed vs Constrained Random
Directed Test (manual, specific scenarios):
task test_basic_write();
  // Engineer manually specifies EVERY stimulus
  @(posedge clk);
  addr   = 32'h0000_1000;
  wdata  = 32'hDEAD_BEEF;
  wr_en  = 1'b1;
  @(posedge clk);
  wr_en  = 1'b0;
  // Check: did data land correctly?
  assert(mem['h1000] == 32'hDEAD_BEEF);
endtask
Constrained Random (automated, vast coverage):
class mem_transaction extends uvm_sequence_item;
  rand bit [31:0]  addr;
  rand bit [31:0]  wdata;
  rand bit          wr_en;
  rand int          delay;

  // Constraints guide randomization
  constraint c_addr {
    addr inside {[32'h0000:32'hFFFF]};
    addr % 4 == 0; // word-aligned
  }
  constraint c_delay {
    delay inside {[0:5]};
  }
  constraint c_write_bias {
    wr_en dist {1 := 70, 0 := 30};
  }
endclass
CRV finds bugs you never thought to test for -- the randomizer explores millions of combinations automatically
■ Coverage Reach: Directed vs Constrained Random

2
Constrained Random — How Constraints Work
The solver picks random values satisfying ALL constraints simultaneously
■ Constraint Block Examples
class pkt_transaction extends uvm_sequence_item;
  rand bit [2:0]   pkt_type;
  rand bit [15:0]  pkt_len;
  rand bit [7:0]   payload[];
  rand bit [3:0]   priority;

  // 1) Range constraint
  constraint c_len {
    pkt_len inside {[64:1518]};  // Ethernet range
  }

  // 2) Conditional (implication) constraint
  constraint c_type_len {
    (pkt_type == 3'b001) ->
        pkt_len inside {[64:128]};
    (pkt_type == 3'b100) ->
        pkt_len inside {[512:1518]};
  }

  // 3) Distribution constraint
  constraint c_priority {
    priority dist {
      0       := 10,  // low  = 10%
      [1:3]   := 20,  // med  = 20%
      [4:7]   := 40,  // high = 40%
      [8:15]  := 30   // crit = 30%
    };
  }

  // 4) Array size constraint
  constraint c_payload {
    payload.size() == pkt_len;
  }
endclass

3
Assertion-Based Verification (SVA)
Properties checked continuously during simulation — catches bugs the instant they happen
■ SVA Code — Protocol Checks
// ── Immediate Assertion ──
always_comb begin
  assert (onehot_bus == 0 ||
         $onehot(onehot_bus))
  else $error("Not one-hot!");
end

// ── Concurrent Assertion: AXI handshake ──
// Once VALID rises, it MUST stay high until READY
property axi_valid_stable;
  @(posedge clk) disable iff (!rst_n)
    $rose(valid) |->
      valid throughout (ready[->1]);
endproperty
assert property (axi_valid_stable)
  else $fatal("VALID dropped before READY!");

// ── Concurrent: req must get ack in 1-5 cycles ──
property req_ack_timeout;
  @(posedge clk) disable iff (!rst_n)
    $rose(req) |-> ##[1:5] ack;
endproperty
assert property (req_ack_timeout);

// ── Cover: did we see back-to-back transfers? ──
cover property (
  @(posedge clk)
    (valid && ready) ##1 (valid && ready)
);
■ SVA Timing — Visual Illustration
Property: $rose(req) |-> ##[1:5] ack;
Green window = valid ack range (1-5 cycles). Top ack passes (T3), bottom ack fails (T6 = too late)

Formal Verification
Mathematically proves properties hold for ALL possible inputs — no testbench needed
■ Formal vs Simulation Approach
┌─────────────────────────────────────────────────┐ │ SIMULATION APPROACH │ └─────────────────────────────────────────────────┘ Testbench ──► Stimulus ──► DUT ──► Checker Tests SPECIFIC scenarios (subset of state space) ┌──────────────────────────────┐ │ ● ● ● ● ● │ ← sample points │ ● ● ● │ │ ● ● │ └──────────────────────────────┘ ┌─────────────────────────────────────────────────┐ │ FORMAL APPROACH │ └─────────────────────────────────────────────────┘ Properties ──► Formal Engine ──► PROVEN or CEX Explores ALL possible states (exhaustive) ┌──────────────────────────────┐ │██████████████████████████████│ ← 100% state space │██████████████████████████████│ │██████████████████████████████│ └──────────────────────────────┘ Result: PROVEN → property holds for ALL inputs ✓ CEX → counter-example waveform found ✗ BOUNDED → proven up to N cycles
■ Formal Property + Counter-Example
// Property: FIFO never overflows
property fifo_no_overflow;
  @(posedge clk) disable iff (!rst_n)
    (wr_en && !rd_en) |->
      (fifo_count < FIFO_DEPTH);
endproperty

// Property: mutual exclusion on bus
property mutex_grant;
  @(posedge clk) disable iff (!rst_n)
    $onehot0(grant);  // at most one grant
endproperty

// Property: no deadlock in arbiter
property no_deadlock;
  @(posedge clk) disable iff (!rst_n)
    (|req) |-> s_eventually (|grant);
endproperty
When formal FAILS → provides Counter-Example (CEX):
Formal tool found: continuous writes without reads causes overflow at depth 8

Coverage-Driven Verification
Measure what you've tested — close coverage holes to reach sign-off
■ Functional Coverage Code
covergroup alu_cov @(posedge clk);

  // Coverpoint: which opcodes were exercised?
  cp_opcode: coverpoint opcode {
    bins add_op  = {4'b0000};
    bins sub_op  = {4'b0001};
    bins mul_op  = {4'b0010};
    bins div_op  = {4'b0011};
    bins shift   = {[4'b0100:4'b0111]};
    bins logic   = {[4'b1000:4'b1011]};
    illegal_bins reserved = {default};
  }

  // Coverpoint: edge cases on operands
  cp_operand_a: coverpoint op_a {
    bins zero     = {0};
    bins max_val  = {'1};  // all 1s
    bins mid      = {[1:254]};
  }

  // Cross coverage: opcode X operand combos
  cx_op_data: cross cp_opcode, cp_operand_a;

endgroup
